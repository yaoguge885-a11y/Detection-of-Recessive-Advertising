## 一、项目整体把握（先建立"大局观"）

### 1.1 一句话定义
构建一个 **LangGraph 驱动的多智能体系统**，输入一条社交媒体内容（文本 + 图片 + 博主历史 + 评论区），自动判定它是 **明广 / 暗广 / 非广**，并输出一份**带证据链和法规依据的、人能看懂的审查报告**。

### 1.2 我们在做的战略转变（务必理解）
申报书里的原始方案是一条**传统流水线**：BERT 提文本特征 → CNN/YOLO 提图像特征 → BERTopic 算主题漂移 → 全部拼接喂给 XGBoost 做二分类。这套方案工程可行，但有三个硬伤：

1. **跨模态语义矛盾抓不住**——"文案卖惨、配图却是完美产品特写"这种图文暗示，向量拼接学不到。
2. **黑箱、不可解释**——XGBoost 只给概率，给不出"为什么判它是广告"的自然语言逻辑链，而合规审查恰恰最需要证据。
3. **规则一变就失效**——平台规则、博主套路天天变，固化流水线训练完知识就锁死了。

因此我们**转向 Agent（智能体）范式**：以大模型为"大脑"，让系统能**自主规划**（该看图还是该查历史）、**调用工具**（把上面那些模型当"手脚"）、**拥有记忆**（记住博主半年的人设）、**多角色协作**（审查员 vs 辩护人辩论）、**引经据典**（RAG 检索法规）。这不是推翻原方案，而是**把原方案里的模型当作智能体的工具**，在其之上加一层会思考、会解释的"指挥系统"。

> ⚠️ 诚实的现实主义：研究报告里的完整蓝图（A2A 协议、四层记忆、ASTRA 红队…）是"理想终态"。4 人 6 个月**不可能全做**。本项目的策略是**实现这套蓝图中一个能落地、能演示、能发论文的切片**，其余作为冲刺目标（见计划表降级阶梯 L2/L3）。

### 1.3 目标架构蓝图（我们要建的系统长什么样）

```
                输入：一条帖子（文本 + 图片 + 博主主页 + 评论区）
                                  │
                    ┌─────────────▼──────────────┐
                    │   Supervisor 主控智能体      │  ← LangGraph 编排：
                    │   规划 / 动态路由 / 汇总      │    纯文本帖就跳过视觉，省算力
                    └──┬──────┬───────┬───────┬────┘
             ┌─────────┘      │       │       └─────────┐
             ▼                ▼       ▼                 ▼
     ┌──────────────┐ ┌────────────┐ ┌──────────┐ ┌──────────────┐
     │ NLP 智能体    │ │ 视觉智能体  │ │ 行为智能体 │ │ RAG 检索      │
     │ 商业意图/情感 │ │ 图文一致/OCR│ │ 记忆/偏移  │ │ 法规库+判例库 │
     │ 话术模式      │ │ Logo/PS 检测│ │ 主题漂移   │ │              │
     └──────┬───────┘ └─────┬──────┘ └────┬─────┘ └──────┬───────┘
            │ 调用工具        │ 调用工具     │ 查向量库      │ 检索条款
            └────────────────┴─────────────┴──────────────┘
                                  │  各自产出「证据 + 置信度」
                                  ▼
                    ┌────────────────────────────┐
                    │  Judge / 聚合智能体          │
                    │  加权聚合 + 反思质询 + 交叉验证│
                    └──────────────┬─────────────┘
                       置信度低 → HITL 人工复核
                                  ▼
             结构化审查报告：判定 + 证据链 + 法规引用 + 置信度
```

对应到 LangGraph 的工程实现：**节点=智能体**，**边=Supervisor 的条件路由**，**工具=被封装的模型**，**State=共享上下文**，**Checkpointer=记忆与持久化**。

### 1.4 技术栈（已于 2026-07-12 在 ADR 定稿，以 `implicit-ad-agent/docs/ADR.md` 为准）

| 层 | 选型 | 理由 |
| --- | --- | --- |
| 编排 | **LangGraph + LangChain**（Python 3.10，本机实际版本，见 ADR-009） | 图式编排、状态、检查点、Supervisor 模板齐全 |
| 大模型 | **厂商无关抽象**（OpenAI 兼容端点）+ **分档用模**：DeepSeek-chat 主力文本、Qwen-VL 视觉、便宜小模型做路由、强推理模型只在低置信分支用（ADR-001） | 通过 LangChain 统一接口，改 `.env` 即换模型。工具调用（Function Call/tool-use）能力是硬指标 |
| 视觉 | 多模态 LLM 优先；PaddleOCR（中文 OCR）；CLIP（图文一致）；YOLOv8（Logo/商品，拓展） | 先用多模态 LLM 快速起步，专用模型逐步替换 |
| 文本 | 中文情感模型 / LLM；BERTopic（主题） | 意图与话术用 LLM，主题分布用 BERTopic |
| 记忆/RAG | **Chroma**（起步，本地零成本）→ Milvus（>100 万向量或需并发服务时再迁）；Embedding 用 **bge-small-zh-v1.5** 本地起步（ADR-003） | 情景记忆 + 法规判例检索；embedding 入库即锁定，中途换要整库重建 |
| 观测 | **LangSmith** | 追踪智能体每一步 Thought/Action/Observation，调试神器 |
| 后端/前端 | FastAPI + **Streamlit/Gradio** | 演示导向，最快出可视化 |
| 工具协议 | 原生 LangGraph tools 起步；**FastMCP** 封装为拓展 | MCP 是研究报告卖点，有余力再上 |

## 二、分阶段指导教学

> 每阶段统一给出：**目标 → 先学什么 → 工作路线 → 工具/库 → 常见坑 → 完成标准**。

### P0 · 启动与技能对齐

- **目标**：全员术语拉齐、环境就绪、跑通第一个 LangGraph 最小图、敲定架构决策书（ADR）。
- **先学**：Agent 与 LLM/Workflow 的区别（见研究报告第 2 节）；LangGraph 三件套 = **State / Node / Edge**；ReAct 循环（Thought→Action→Observation）。
- **工作路线**：
  1. 每人精读 5 份资料中与自己主责相关的部分，产出术语笔记。
  2. L 主持写 **ADR**（架构决策记录）：LLM 选型、向量库、前端、降级阶梯——白纸黑字，避免后期反复。
  3. 搭环境：虚拟环境、`requirements.txt`、`.env`（API Key）、Git 仓库、LangSmith 项目。
  4. 照官方文档跑通 "ReAct + 1 个自定义工具" 示例，在 LangSmith 里看到调用轨迹。
- **工具/库**：LangGraph、LangChain、`python-dotenv`、LangSmith、Git。
- **常见坑**：一上来就想搭多智能体——**别**，先把单节点+单工具跑通；API Key 直接写进代码提交到仓库（用 `.env` + `.gitignore`）。
- **完成标准（M0）**：ADR 定稿、环境全员可跑、Hello-LangGraph 有轨迹。

### P1 · 数据地基与标注规范

- **目标**：拿到一份**高质量、三元标签、标注一致**的种子数据集。这是全项目的地基，地基歪了后面全塌。
- **先学**：明广/暗广/非广的判定学理（申报书研究现状 + 那篇印尼原生广告数据集的"情感/劝服/产品/视角"四类隐性特征）；标注一致性指标 Cohen's κ。
- **工作路线**：
  1. **合规先行**：先查各平台 ToS 与爬虫红线，优先用公开数据集 + 有限的、克制的、只读的采集 + 人工收集；记录在 `data_compliance.md`。
  2. 定 **Schema**：帖子（正文/图片/时间）、博主（历史发文/资料）、评论区（评论/互动）。
  3. 写 **标注规范**：给"暗广"下可操作定义（如：未标注但有明确导购意图、伪造截图、置顶产品型号…），并配 ≥20 个**边界案例**（生活分享 vs 恰饭的模糊地带）。
  4. **双人独立标注 + 第三人仲裁**，算 κ；κ<0.6 就回炉改规范。
  5. 划分 train/dev/test，保证**三集之间博主不重叠**（防数据泄漏）。
- **工具/库**：采集脚本、`pandas`、`label-studio`（可选标注平台）、`scikit-learn`（算 κ）。
- **常见坑**：标签定义含糊导致 κ 上不去（→ 多写边界案例）；同一博主的帖子横跨训练/测试集（→ 按博主切分）；只标暗广不标明广/非广（→ 三元才有对比价值）。
- **完成标准（M1）**：≥1500 条三元标签数据，κ≥0.6，规范与仲裁记录齐全。

### P2 · 工具舱：把模型封装成"手脚"

- **目标**：把每个分析能力做成**输入输出规范、可独立调用、能容错**的工具，供智能体按需调用。对应研究报告"存量资产盘活"。
- **先学**：LangChain 的 `@tool` 装饰器与工具 schema；Function Call 底层机制（LLM 生成结构化 JSON→引擎拦截执行→回填结果）；结构化输出（Pydantic）。
- **工作路线**：每个工具都遵循同一套骨架 —— 明确的入参、明确的出参（含**证据字段**）、超时与空值兜底、一个单测。清单见计划表 P2。
  - 起步策略：**能用多模态 LLM 就先用 LLM**（图文一致、Logo 识别、意图判断都可由多模态 LLM 直接给结构化结果），把 YOLO/BERTopic 等自训练模型作为后续替换项；只要**接口不变**，替换是无痛的。
- **工具/库**：LangChain tools、PaddleOCR、CLIP/多模态 LLM、BERTopic、`scipy`（KL/JS 散度）、`pytest`。
- **常见坑**：工具直接把整张图/长文塞进上下文导致 token 爆炸（→ 先摘要/裁剪）；工具没有容错，遇到无图帖子直接抛异常拖垮整图（→ 每个工具都要有 graceful fallback）；证据字段缺失导致后面报告无法解释（→ 每个工具都必须返回"依据"）。
- **完成标准（M2）**：≥6 工具单测全绿、异常不崩、各自返回证据。

### P3 · 单体智能体 MVP（**保底成果，务必拿下**）

- **目标**：一个 **ReAct 单体智能体**端到端跑通：读帖子→自主调工具→查 RAG 法规→输出结构化报告。**完成这一步，项目就已经"可答辩、可发论文"了。**
- **先学**：LangGraph 的 `create_react_agent` / 自建 StateGraph；`with_structured_output` 强约束输出；RAG 基本流程（切块→向量化→检索→注入上下文）。
- **工作路线**：
  1. 设计全局 **State**：`post`（输入）、`evidence`（各工具证据累积）、`verdict`（明/暗/非广）、`confidence`、`report`。
  2. 用 ReAct 让智能体自主决定调哪些工具、调几次。
  3. 建 **RAG 法规库**：录入《互联网广告管理办法》、各平台商业推广规范、历史判例，检索节点把相关条款注入。
  4. 用 Pydantic 定义**审查报告**结构：判定 + 逐条证据 + 法规引用 + 置信度。
  5. **评估**：在 test 集跑 P/R/F1，并**复现一个 XGBoost 流水线作对照基线**（这正好用得上申报书原方案，做成论文里的 baseline）。
  6. 开始写论文的引言/相关工作/方法。
- **工具/库**：LangGraph、Chroma、Pydantic、XGBoost（基线）、`scikit-learn`。
- **常见坑**：ReAct 陷入死循环反复调同一工具（→ 设最大步数 + 反思）；LLM 输出不守格式（→ 按 ADR-010：json_mode + 英文字段名 + Pydantic 校验 + 规则降级，国产端点别用默认 function-calling 结构化模式）；RAG 检索回来一堆无关条款（→ 调 chunk 大小与 top-k，加重排序）。
- **完成标准（M3）**：端到端出结构化报告 + 首版 P/R/F1 + XGBoost 对照。

### P4 · 多智能体协作 + 记忆（项目"多智能体"卖点核心）

- **目标**：从单体裂变为 **Supervisor + 专家 + Judge** 的协作网络，并接入**博主长期记忆**与**偏好偏移**检测。
- **先学**：Supervisor 路由模式（`langgraph-supervisor`）；MemGPT 的分层记忆思想；情景记忆 vs 语义记忆；EMA（指数移动平均）算偏移；反思/多智能体辩论。
- **工作路线**：
  1. **Supervisor**：根据输入动态路由——纯文本帖跳过视觉智能体，有图有历史才全开。
  2. **专家智能体**：NLP（意图/情感/话术）、视觉（图文一致/OCR/Logo）、行为（记忆/主题漂移）。评论区智能体列为拓展。
  3. **情景记忆**：把博主历史发文的语义向量存进向量库；处理新帖时检索"该博主过去是否提过同类产品/表达过相反态度"。
  4. **语义画像 + 偏好偏移**：把博主抽象成稳定人设（如"性价比学生党美妆博主"），当前帖若剧烈背离，用滑窗 + EMA 算出**偏移分**作为暗广的核心证据。
  5. **Judge/聚合**：借鉴 2508.10143 的**加权聚合**（按各 agent 可靠度加权），低置信触发**反思质询**（"文案痛点和图片焦点一致吗？""历史人设和当前推荐断裂吗？"）。
  6. **HITL**：置信度低于阈值就挂起转人工。
  7. **消融实验**：文本 → +视觉 → +行为 → 全量，看每一层的增益。
- **工具/库**：`langgraph-supervisor`、Chroma/Milvus、`numpy`（EMA）、LangSmith。
- **常见坑**：智能体太多导致上下文互相干扰、成本飙升（→ 控制在 Supervisor+3 专家+Judge）；记忆无限增长（→ 摘要压缩 + 时间/相似度检索，别全量塞）；辩论收敛不了（→ 限定轮数，Judge 最终裁决）。
- **完成标准（M4）**：多智能体+记忆版跑通，**暗广召回率优于 XGBoost 基线**，有消融表。

### P5 · 系统集成 · 前后端 · 评估

- **目标**：把智能体图包成可现场演示的 Web 应用，产出完整可入论文的评估。
- **先学**：FastAPI 基础；Streamlit/Gradio 快速前端；评估指标体系（尤其 AUC-ROC 与消融）；提示词注入防护。
- **工作路线**：
  1. FastAPI 暴露 `/analyze`：输入链接/帖子 → 返回 JSON 报告。
  2. Streamlit/Gradio 做前端：输入框 + 证据链高亮 + 法规引用展示。
  3. **完整评估**：P/R/F1（重点看暗广召回）、AUC-ROC、分类别、消融、**可解释性人评**（证据链是否服人）。
  4. **基础安全**：过滤图片/文本里的注入指令（"忽略以上指令…"）；工具保持只读、不越权。
  5. 论文初稿定型，补齐实验图表。
- **工具/库**：FastAPI、Streamlit/Gradio、`matplotlib`/`seaborn`。
- **常见坑**：演示当场调实时爬虫翻车（→ 预置缓存样本兜底）；只报准确率不报召回（→ 暗广是少数类，召回和 F1 才是关键）。
- **完成标准（M5）**：Web 可演示 + 完整评估 + 论文初稿。

### P6 · 论文 · 竞赛 · 结题

- **目标**：把成果变现为论文、竞赛、软著、结题。
- **工作路线**：论文定稿投稿 → 竞赛材料与演示视频 → 软著申请 → 结题表与答辩 PPT → 代码归档可复现。
- **常见坑**：论文全堆到最后写（→ 从 P3 起滚动写）；演示脚本没排练（→ 提前跑通固定 demo 流程）。
- **完成标准（M6）**：论文定稿、参赛包、软著、结题包齐全。

## 三、数据与标注规范要点（P1 展开）

- **三元标签**：明广（有明确标识/赞助字样）、暗广（无标识但有明确导购意图/伪造/水军）、非广（正常分享）。
- **暗广判定信号**（写进规范）：制造焦虑的话术、"亲身经历"式软文、模糊商业词、图文暗示（卖惨配产品特写）、伪造销量截图、置顶产品型号、评论区水军格式化赞美、与历史人设突变。
- **一致性**：双标 + 仲裁，Cohen's κ≥0.6 才可用；边界案例越多，κ 越稳。
- **防泄漏**：按博主切分 train/dev/test。
- **数据增益**：可用假新闻/原生广告公开数据集做迁移预热，缓解标注稀缺。

## 四、评估方案与论文骨架

**评估**
- 主指标：Precision / Recall / **F1**（重点关注**暗广召回**，因其为少数类且漏报代价高）、AUC-ROC、分类别指标。
- **消融**：文本 → +视觉 → +行为 → +评论 → 全量，证明每一模态/智能体的贡献。
- **对照**：本智能体系统 vs 申报书 XGBoost 传统融合基线。
- **可解释性**：人评证据链质量（是否让审查员信服）。

**论文骨架（从 P3 起滚动填充）**
1. Introduction：隐性广告治理的空白与可解释性诉求。
2. Related Work：广告理解、假新闻多模态检测、Agentic AI（引 2508.10143 / 2603.20351 / MemGPT / 两篇假新闻）。
3. Method：多智能体架构、工具舱、记忆与偏好偏移、加权聚合。
4. Dataset：三元数据集与标注规范、κ。
5. Experiments：主结果 + 消融 + 与 XGBoost 对照 + 可解释性。
6. Discussion / Limitation / Ethics：合规、注入防护、HITL。

## 七、关键词清单（团队学习地图 · 建议逐个打勾）

> 用法：把它当"技能树"。每个词能**一句话讲清 + 知道在本项目哪儿用**，才算掌握。

### A. 智能体与编排
- [ ] **Agent（智能体）**：以 LLM 为大脑、含感知/记忆/规划/工具的自治体。本项目的系统本体。
- [ ] **LangGraph**：图式智能体编排框架（State/Node/Edge）。本项目主干。
- [ ] **LangChain**：LLM 应用工具链，提供模型/工具统一接口。
- [ ] **ReAct**：Thought→Action→Observation 交替循环。单体智能体的工作模式。
- [ ] **Plan-and-Solve**：先出全局计划再分步执行。长程任务用。
- [ ] **Reflection / 自我纠错**：产出结论后内部批评再修正。降低误判。
- [ ] **Multi-Agent Debate（多智能体辩论）**：审查员 vs 辩护人对抗+仲裁。高主观场景提置信度。
- [ ] **Supervisor 模式**：主控智能体动态路由/汇总。本项目的调度中枢。
- [ ] **State（状态）**：图中流转的共享上下文。存证据/结论/置信度。
- [ ] **Checkpointer**：状态持久化/记忆落盘机制。
- [ ] **LangSmith**：智能体运行追踪与评测。调试核心工具。

### B. 工具与协议
- [ ] **Function Call / Tool Use**：LLM 生成结构化 JSON 指令调用外部函数。工具调用的底层机制。
- [ ] **Tool（工具）**：封装好的确定性能力（OCR、检测…）。智能体的"手脚"。
- [ ] **Skill vs Prompt**：Skill=确定性代码能力；Prompt=自然语言指令。别混淆。
- [ ] **MCP（Model Context Protocol）**：Anthropic 提出的"AI 的 USB-C"，标准化智能体↔工具通信。本项目拓展项（FastMCP 封装）。
- [ ] **FastMCP**：Python 快速搭 MCP Server 的框架。
- [ ] **A2A（Agent-to-Agent）**：智能体间横向协作协议（Agent Card 广播能力）。冲刺项。
- [ ] **结构化输出 / Pydantic**：强约束 LLM 输出为固定 schema。保证报告可解析。

### C. 记忆与检索
- [ ] **MemGPT / 虚拟上下文管理**：分层记忆、像 OS 分页一样调度上下文。记忆系统的理论支撑。
- [ ] **工作/情景/语义/程序记忆**：四层记忆。本项目重点用**情景**（博主历史）与**语义**（人设画像）。
- [ ] **向量数据库（Chroma/Milvus/Pinecone）**：存嵌入、按相似度检索。情景记忆与 RAG 的底座。
- [ ] **Embedding（嵌入）**：把文本/图像变成向量。检索的基础。
- [ ] **RAG（检索增强生成）**：先检索外部知识再生成。用于**法规/判例**引用。
- [ ] **偏好偏移（Preference Shift）/ EMA**：量化博主人设突变。暗广核心证据。
- [ ] **主题漂移（Topic Drift）/ KL·JS 散度**：量化发文主题变化。行为信号。

### D. 多模态与模型
- [ ] **多模态（Multimodal）**：文本+图像+行为+评论联合分析。项目底色。
- [ ] **BERT / RoBERTa**：预训练语言模型。文本意图/情感（或直接用 LLM）。
- [ ] **CLIP**：图文对齐模型。**图文一致性**检测。
- [ ] **OCR / PaddleOCR**：图像取文字/二维码/销量截图。
- [ ] **YOLO / ResNet**：目标检测/图像特征。Logo/商品识别（拓展）。
- [ ] **BERTopic / LDA**：主题建模。博主内容图谱。
- [ ] **GNN（图神经网络）**：建模评论区互动网络、发现水军社群。拓展项。
- [ ] **跨模态注意力 / 对比学习 / 混合注意力**：两篇假新闻论文的融合技术。可做图文一致性工具。
- [ ] **XGBoost / LightGBM**：梯度提升树。作为**传统基线**对照。
- [ ] **多模态大模型（VLM）**：能直接看图+读文出结论。起步期视觉工具首选。

### E. 领域与治理
- [ ] **隐性广告 / 暗广 / 软广 / 恰饭 / 种草**：无标识但有导购意图的推广。识别目标。
- [ ] **明广 / 暗广 / 非广**：三元标签体系。
- [ ] **Product Placement / Native Advertising**：植入式/原生广告。国际对应概念。
- [ ] **Ad Fraud（广告欺诈）**：刷量/点击欺诈。与本项目相邻但不同。
- [ ] **KOL / 商单**：意见领袖 / 商业合作贴。数据来源场景。
- [ ] **可解释性（Explainability）/ 证据链**：给出"为什么判它是广告"。本项目最大卖点。

### F. 安全与合规
- [ ] **提示词注入（Prompt Injection）**：恶意内容劫持 LLM。需防护。
- [ ] **HITL（人在回路）/ 审查熔断**：低置信转人工。负责任 AI 底线。
- [ ] **RBAC / 纯函数工具 / 沙箱**：多智能体身份控制、工具只读隔离。
- [ ] **ASTRA / 红队测试**：对抗性安全评估。冲刺项。

### G. 数据与评估
- [ ] **标注规范 / 边界案例**：暗广判定细则。地基。
- [ ] **Cohen's κ / 标注一致性**：衡量标注可靠度（门槛 0.6）。
- [ ] **Precision / Recall / F1**：核心指标，**重点看暗广召回**。
- [ ] **AUC-ROC**：整体判别力。
- [ ] **消融实验（Ablation）**：逐层验证各模态贡献。
- [ ] **数据泄漏 / 按博主切分**：防训练测试串味。

### H. 工程与交付
- [ ] **FastAPI / Streamlit / Gradio**：后端接口 / 快速演示前端。
- [ ] **ADR（架构决策记录）**：白纸黑字锁定关键选型。
- [ ] **软件著作权 / 结题答辩**：申报书承诺的交付形式。

## 八、给团队的 5 条心法

1. **先跑通，再变漂亮**：每阶段结束都要有一个能演示的版本；宁可少而完整。
2. **保底成果在 P3**：单体智能体 MVP 一旦拿下，项目就"不会翻车"，其余都是加分。
3. **文档随做随写**：论文/说明从 P3 滚动记录，别堆到期末。
4. **用 LangSmith 看轨迹**：智能体出问题先看轨迹，别靠猜。
5. **合规与诚信是红线**：数据采集守 ToS，标注实事求是，成果不注水。
