> **LightRAG 参考边界**：必须先交付 Chroma 法规 RAG 基线，再在独立环境中通过 REST/薄适配器试验 LightRAG；不得直接改写 LangGraph 主干，也暂不用于 P4 博主历史记忆。对照集固定为 30 问（10 条直接条款、10 条跨文档关系、10 条应拒答），记录 Recall@5、引用准确率、无答案误引、P95 延迟和索引成本。
> **Go 门槛**：Recall@5 不低于 Chroma、引用准确率 ≥95%、跨文档题至少比 Chroma 多答对 2/10、无答案误引 ≤1/10、P95 延迟 ≤2 倍 Chroma；任一核心门槛不满足就保留 Chroma，不把 LightRAG 纳入主线。
> 官方学习入口：[仓库与 README](https://github.com/HKUDS/LightRAG)、[API Server](https://github.com/HKUDS/LightRAG/blob/main/docs/LightRAG-API-Server.md)、[SDK/QueryParam](https://github.com/HKUDS/LightRAG/blob/main/docs/ProgramingWithCore.md)、[自定义知识图谱示例](https://github.com/HKUDS/LightRAG/blob/main/examples/insert_custom_kg.py)。

## 6. MVP 优先级与降级阶梯（进度落后时按此砍功能，永不"翻车"）

| 层级 | 内容 | 说明 |
| --- | --- | --- |
| **L0 必做（保底）** | 单体 ReAct 智能体 + 文本&图像(多模态LLM) + RAG 法规 + 结构化报告 + 三元评估 + Web demo | 完成 L0 即为**完整可答辩、可发论文**的项目 |
| **L1 应做** | Supervisor + 3 专家 + Judge；情景记忆 + 偏好偏移；加权聚合 | 本项目"多智能体"卖点的核心 |
| **L2 可做** | 反思/辩论；评论区水军(GNN/规则)；图像取证；HITL 界面；FastMCP 封装 | 有余力再上 |
| **L3 冲刺** | A2A 协议；程序记忆/自我优化；ASTRA 红队；跨平台矩阵 | 面向获奖/顶会的加分项 |

> 判断准则：**每个阶段结束都必须有一个"能跑、能演示"的版本**。宁可功能少而完整，不要功能多而半残。

**降级决策点**（详见 ADR-008；关口日期一到当场决策，不拖"再等一周"）：

| 检查点 | 日期 | 未达标时的动作 |
| --- | --- | --- |
| M1 | 08-17 | κ<0.6 或数据不足 → P2 压缩到 2 周（砍 2.5/2.7），时间还给数据 |
| M3 | 10-05 | MVP 未跑通 → 冻结 L1 范围：P4 只做 Supervisor 路由 + 情景记忆 |
| M4 | 11-16 | 暗广召回未超基线 → 停止加智能体，P5 全力调优与评估 |
| M5 | 12-14 | 评估不完整 → 论文降级为"系统 + 初步实验"，保结题与软著 |

**升级准则**：当前层级全部完成且里程碑提前，才允许动下一层；不得跳层"顺手做 L3"。
